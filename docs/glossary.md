# Глоссарий терминов

Этот файл содержит определения терминов, используемых в проекте. **Важно**: при введении новых терминов обязательно добавляйте их в этот файл для синхронизации понимания сущностей.

## Основные термины

### Operator (Оператор)
Kubernetes оператор - это паттерн расширения Kubernetes, который использует Custom Resources (CR) для управления приложениями и их компонентами. Оператор следует принципам Kubernetes, в частности концепции управления через декларативные API.

### Custom Resource (CR)
Расширение Kubernetes API, которое не является частью стандартной установки Kubernetes. Позволяет определять собственные типы ресурсов.

### Custom Resource Definition (CRD)
Определение схемы для Custom Resource. CRD описывает структуру данных, валидацию и версионирование кастомного ресурса.

### Controller (Контроллер)
Компонент оператора, который отслеживает изменения ресурсов и выполняет действия для приведения текущего состояния к желаемому (reconciliation loop).

### Reconciliation Loop (Цикл согласования)
Процесс, при котором контроллер постоянно проверяет текущее состояние ресурсов и сравнивает его с желаемым состоянием, внося необходимые изменения.

### cert-manager
Kubernetes оператор для автоматического управления TLS сертификатами. Используется для получения и обновления сертификатов от различных CA (Certificate Authority), включая Let's Encrypt.

### Let's Encrypt
Бесплатный, автоматизированный и открытый центр сертификации (CA), предоставляющий SSL/TLS сертификаты.

### HTTP01 Challenge
Метод проверки домена (domain validation) для Let's Encrypt, при котором проверяющий сервер делает HTTP запрос на домен по пути `/.well-known/acme-challenge/<token>`. cert-manager создает временные поды для обработки этих запросов.

### ACME (Automated Certificate Management Environment)
Протокол, используемый Let's Encrypt для автоматизации выдачи сертификатов. HTTP01 - один из методов challenge в рамках протокола ACME.

### Pod
Наименьшая единица развертывания в Kubernetes. Представляет один или несколько контейнеров, которые работают вместе и разделяют ресурсы.

### Challenge Pod
Временный под, создаваемый cert-manager для обработки HTTP01 challenge запросов от Let's Encrypt. Этот под должен быть доступен по HTTP на домене, для которого запрашивается сертификат.

### Domain Validation (Валидация домена)
Процесс подтверждения права владения доменом перед выдачей SSL/TLS сертификата. Let's Encrypt использует различные методы challenge для валидации.

### Certificate (Сертификат)
SSL/TLS сертификат, используемый для шифрования соединений. В контексте Kubernetes - это Kubernetes ресурс, управляемый cert-manager.

### CertificateRequest
Kubernetes ресурс, создаваемый cert-manager для запроса сертификата от CA. Содержит CSR (Certificate Signing Request) и метаданные.

### Reconcile (Согласование)
Процесс приведения текущего состояния системы к желаемому состоянию, описанному в Custom Resource.

### Finalizer
Механизм Kubernetes, позволяющий контроллеру выполнить очистку перед удалением ресурса. Ресурс с finalizer не может быть удален до тех пор, пока finalizer не будет удален.

### RBAC (Role-Based Access Control)
Система контроля доступа в Kubernetes, основанная на ролях. Оператор должен иметь соответствующие права для работы с ресурсами.

### Kustomize
Инструмент для настройки Kubernetes конфигураций. Используется Operator SDK для генерации манифестов.

### EnvTest
Тестовое окружение для контроллеров, предоставляющее реальный Kubernetes API server без необходимости полного кластера.

## Термины Operator SDK

### Operator SDK
Инструментарий для создания, тестирования и развертывания Kubernetes операторов. Поддерживает Ansible, Helm и Go операторы.

### Controller Runtime
Библиотека для создания контроллеров Kubernetes. Основа для Go операторов в Operator SDK.

### Manager
Компонент controller-runtime, который управляет жизненным циклом контроллеров и обеспечивает их работу.

### Watch
Механизм отслеживания изменений ресурсов в Kubernetes. Контроллеры используют watches для получения уведомлений об изменениях.

### Predicate
Фильтр для событий watch, позволяющий контроллеру обрабатывать только определенные изменения ресурсов.

### Webhook
HTTP сервер, который обрабатывает запросы от Kubernetes API server. Используется для валидации и мутации ресурсов.

## Термины проекта

### CertMonitor
Custom Resource, определяемый этим оператором для мониторинга cert-manager подов HTTP01 challenge.

### HTTP01 Pod Monitor
Компонент оператора, который отслеживает состояние подов, созданных cert-manager для HTTP01 challenge.
